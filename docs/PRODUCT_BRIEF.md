# Product Brief

## A. 목적

사용자가 옷을 언제 입고 빨았는지 기억하지 않아도 되도록, 모바일 웹에서 마지막 세탁일과 착용 횟수를 빠르게 기록하고 소재·용도별 세탁 필요 상태를 확인하는 서비스입니다.

## B. 핵심 판단

| 항목 | 판단 |
|---|---|
| 1차 경험 | 빠른 문장 입력과 카드 액션으로 `오늘 입음`, `세탁 완료`, `땀남/냄새/얼룩`을 바로 기록 |
| 우선 플랫폼 | 설치 없는 모바일 웹 |
| 초기 사용자 | 운동복과 출근복을 반복 착용하는 1인 직장인 |
| MVP 목표 | 기록 부담을 줄이고 세탁 필요 여부와 이유를 한 화면에서 판단 |
| 차별점 | 옷감과 용도에 따라 세탁/통풍/관리 권장을 다르게 제안 |
| 저장 기준 | 데모 성공은 LocalStorage 기준, Supabase는 시간이 남으면 연결 |

## C. MVP 범위

| 기능 | 설명 | 우선순위 |
|---|---|---|
| 옷 등록 | 이름과 종류만 입력 | P0 |
| 마지막 세탁일 표시 | 며칠 전 빨았는지 표시 | P0 |
| 세탁 완료 기록 | 오늘 날짜로 갱신하고 착용 횟수 초기화 | P0 |
| 착용 기록 | 착용 횟수를 늘려 세탁 권장 판단에 반영 | P0 |
| 소재 선택 | 면, 폴리에스터, 울, 데님, 실크 등 기본 소재 선택 | P0 |
| 상태 라벨 | 깨끗함, 곧 세탁, 세탁 권장, 통풍 권장으로 구분 | P0 |
| 권장 이유 | `땀 기록`, `소재 특성`, `착용 횟수` 등 짧게 표시 | P0 |
| 상태 칩 | `땀남`, `냄새`, `얼룩`을 권장 로직에 우선 반영 | P0 |
| 빠른 자연어 입력 | `오늘 러닝 반팔 입었어`, `흰 셔츠 세탁했어` 수준의 mock 파싱 | P0-lite |
| 계정/동기화 | DB 도입 후 구현 | P2 |

## D. 보류 범위

텔레그램 봇, 다채널 IM, 세탁기 연동, 의류 이미지 인식, Cost Per Wear, 수익모델, 가족 공유는 MVP 이후로 둡니다. PRD에서 중요도가 높게 나오면 P1로 재평가합니다.
