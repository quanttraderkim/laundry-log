# WashLog PRD

## A. 핵심 요약

WashLog는 사용자가 옷을 마지막으로 언제 입고 빨았는지 기억하지 않아도 되도록, 모바일 웹에서 착용과 세탁/관리를 3초 안에 기록하고 옷별 세탁 필요 상태를 바로 판단하게 하는 서비스입니다.

| 항목 | 결정 |
|---|---|
| 핵심 문제 | 세탁 시점을 기억에 의존해 불필요하게 자주 빨거나, 반대로 필요한 옷을 늦게 빰 |
| 1차 사용자 | 운동복과 출근복을 반복 착용하는 1인 직장인 |
| 해커톤 차별점 | `말하듯이 기록` + `소재/상태별 세탁 권장 이유`를 첫 화면에서 제안 |
| 1차 플랫폼 | 설치 없는 모바일 웹 |
| 데모 목표 | `문장 입력 → 상태 칩 선택 → 권장 이유 확인 → 세탁 완료` 흐름을 첫 화면에서 완료 |
| 저장 기준 | 데모 성공은 LocalStorage 기준. Supabase는 시간이 남으면 연결 |
| 제외 범위 | 텔레그램 봇, 다채널 IM, 이미지 인식, 수익모델, Cost Per Wear, 장기 인사이트 |

## B. 배경과 판단

기존 `wardrobe-service-plan`의 핵심은 “말하듯이 기록한다”입니다. 다만 원문은 텔레그램 봇, AI 파싱, OAuth, 상세 타임라인, 인사이트, 수익모델까지 포함한 12주형 기획이므로 해커톤 범위로 그대로 가져오면 과합니다.

이번 PRD에서는 봇을 만들지 않고 모바일 웹 안에 빠른 기록 입력을 둡니다. 사용자는 버튼으로 바로 기록하거나 `오늘 러닝 반팔 입었어`, `흰 셔츠 세탁했어`처럼 짧게 입력할 수 있습니다. 자연어 입력은 데모용 P0-lite로 두고, 2~3개 문장 패턴만 룰 기반/mock 파싱합니다. 실제 LLM 파싱은 후순위로 둡니다.

## C. 사용자 가치

| 사용자 상황 | 불편 | WashLog 제공 가치 |
|---|---|---|
| 운동복을 자주 입음 | 땀 냄새가 남기 쉬운데 매번 기억하기 어려움 | 착용 기록만 누르면 세탁 권장으로 올라감 |
| 니트·울 의류를 관리함 | 너무 자주 빨면 옷이 상함 | 세탁 대신 통풍/관리 권장으로 안내 |
| 데님·아우터를 반복 착용함 | 며칠 전 빨았는지 애매함 | 마지막 세탁일과 착용 횟수를 함께 표시 |
| 냄새 판단이 어려움 | 감각에 의존하면 놓치기 쉬움 | 소재/용도 기준으로 보수적 알림 제공 |

| 기능 | 설명 | 우선순위 |
|---|---|---|

| 옷 목록 상태 | 마지막 세탁일, 착용 횟수, 상태 라벨, 권장 이유 표시 | P0 |
| 오늘 입음 | 착용 횟수 증가, 상태 즉시 재계산 | P0 |
| 세탁 완료 | 마지막 세탁일 갱신, 착용 횟수 초기화 | P0 |
| 소재별 권장 규칙 | 속옷/운동복/폴리에스터/울/데님 등 기본 규칙 적용 | P0 |
| 상태 칩 | `땀남`, `냄새`, `얼룩` 입력을 권장 로직에 우선 반영 | P0 |
| 빠른 기록 입력 | `오늘 OOO 입었어`, `OOO 세탁했어` 수준 mock 파싱 | P0-lite |
| 관리 방식 선택 | 세탁기/손세탁/드라이/통풍/스타일러 중 선택 | P1 |
| 계정/동기화 | Supabase 도입 후 사용자별 저장 | P2 |

## E. 세탁 권장 로직

세탁 권장은 절대 규칙이 아니라 “기본 제안”으로 표현합니다. 얼룩, 냄새, 땀, 비·음식 냄새처럼 사용자가 직접 입력한 상태는 소재 규칙보다 우선합니다.

| 옷/소재 | 기본 권장 | 앱 표현 |
|---|---|---|
| 속옷, 양말, 이너웨어 | 매 착용 후 세탁 | `오늘 세탁 권장` |
| 운동복, 땀 많은 착용 | 매 착용 후 세탁 | `땀 기록 있음` |
| 폴리에스터/합성섬유 운동복 | 냄새가 남기 쉬워 빠른 세탁 권장 | `소재 특성상 세탁 권장` |
| 면 티셔츠, 민소매 | 피부 접촉이 크면 매 착용 후 세탁 | `피부 접촉 의류` |
| 셔츠, 슬랙스, 일반 바지 | 2~3회 착용 후 점검 | `곧 세탁` |
| 데님 | 약 3회 착용 후 점검 | `착용 횟수 기준 점검` |
| 울, 캐시미어, 니트 | 잦은 세탁보다 통풍/부분 관리 우선 | `통풍 후 재착용 가능` |
| 실크, 흰옷 | 변색·오염 우려가 크면 빠른 관리 | `오염 확인 필요` |

상태 계산은 아래 순서로 처리합니다.

1. 얼룩/강한 냄새/땀 많음이 있으면 `세탁 권장`
2. 소재·종류별 착용 횟수 기준을 넘으면 `세탁 권장`
3. 기준의 70% 이상이면 `곧 세탁`
4. 울/니트류는 세탁 기준 전까지 `통풍 권장`을 우선 노출
5. 그 외는 `깨끗함`

세탁과 관리는 상태 계산에서 분리합니다. `washed`, `dry_cleaned`는 위생 기준을 초기화하지만, `aired`, `styler`는 냄새/구김 완화 기록으로만 보고 세탁 이후 착용 횟수를 초기화하지 않습니다.

## F. 화면 요구사항

첫 화면은 목록형 대시보드입니다. 사용자가 앱을 열자마자 어느 옷이 세탁 필요인지 알 수 있어야 하며, 각 카드에는 이름, 종류/소재, 마지막 세탁일, 착용 횟수, 상태 라벨, 권장 이유가 표시됩니다.

| 화면 요소 | 요구사항 |
|---|---|
| 상단 요약 | 등록한 옷 수, 세탁 권장 수, 오늘 기록 수 |
| 빠른 기록 | `오늘 러닝 반팔 입었어`, `흰 셔츠 세탁했어` mock 파싱 |
| 빠른 등록 | 이름, 종류, 소재 선택 후 추가 |
| 필터 | 전체, 관리 필요, 깨끗함 |
| 옷 카드 | 상태 라벨과 이유를 한 줄로 표시 |
| 주요 액션 | `오늘 입음`, `세탁 완료` |
| 상태 칩 | `땀남`, `냄새`, `얼룩`을 한 번에 기록 |
| 선택 액션 | P1에서 `통풍`, `드라이`, `스타일러` 추가 |

## G. 데이터 요구사항

옷 데이터에는 소재와 기본 관리 규칙을 저장합니다. 이벤트는 착용, 세탁, 관리, 자연어 입력 결과를 모두 같은 로그로 남기는 편이 확장에 유리합니다.

| 데이터 | 필수 필드 |
|---|---|
| clothing_items | `id`, `user_id`, `name`, `category`, `material`, `care_profile`, `created_at` |
| clothing_events | `id`, `clothing_item_id`, `event_type`, `event_date`, `care_method`, `condition_flags`, `source`, `raw_input`, `note` |
| care_rules | 해커톤에서는 코드 상수, 이후 DB 테이블화 |

## H. 수용 기준

| 기준 | 성공 조건 |
|---|---|
| 기록 속도 | 사용자가 옷 카드에서 3초 안에 착용/세탁 기록 가능 |
| 판단 가능성 | 각 옷의 상태와 권장 이유가 첫 화면에서 보임 |
| 소재 반영 | 같은 착용 횟수라도 운동복/울/데님이 다른 상태로 계산됨 |
| 모바일 적합성 | 390px 폭에서 입력창, 필터, 카드 버튼이 겹치지 않음 |
| 해커톤 데모 | 네 가지 샘플 옷으로 소재별 추천 차이가 설명 가능 |

### H-1. 데모 상태 전이 기준

| Given | When | Then |
|---|---|---|
| 폴리에스터 러닝 반팔이 세탁 후 0회 착용 | `오늘 러닝 반팔 입었어` 입력 | `세탁 권장`, 이유 `폴리에스터 운동복은 1회 착용 후 세탁` |
| 러닝 반팔 카드에서 상태 칩 선택 | `땀남` 선택 | `세탁 권장`, 이유 `땀/냄새 기록 있음` |
| 울 니트가 세탁 후 1회 착용 | `오늘 입음` 기록 | `통풍 권장`, 이유 `울 소재는 세탁보다 통풍 우선` |
| 데님 팬츠가 세탁 후 3회 착용 | 목록 진입 | `곧 세탁`, 이유 `데님 3회 착용, 상태 점검` |
| 흰 셔츠가 관리 필요 상태 | `흰 셔츠 세탁했어` 입력 | `깨끗함`, 착용 횟수 0회, 상태 칩 초기화 |

## I. 보류와 결정 필요

| 항목 | 현재 판단 |
|---|---|
| 서비스명 | 레포는 `laundry-log`, 화면명은 `WashLog` 유지. 최종명은 팀 합의 필요 |
| DB 도입 시점 | 데모 성공은 LocalStorage 기준. Supabase는 시간이 남으면 연결 |

| 알림 | 해커톤에서는 화면 상태로 대체. 푸시/메일/봇 알림은 제외 |
| 소재 목록 | `면/폴리에스터/울/데님/실크/혼방/기타`부터 시작 |

## J. 근거

American Cleaning Institute는 세탁 여부가 활동, 옷감, 착용 상태, 날씨에 따라 달라진다고 보고, 속옷·양말·티셔츠는 매 착용 후, 청바지는 보통 3회 착용 후 세탁을 권장합니다. PubMed에 등재된 운동 후 티셔츠 연구에서는 폴리에스터가 면보다 냄새가 더 강하고 불쾌하게 평가됐습니다. Woolmark는 울이 냄새·오염·구김에 강해 덜 자주 세탁해도 된다고 설명합니다.
